# Clothing Recognition Application - Обзор проекта

## Описание
Приложение для распознавания одежды на изображениях с использованием ИИ (AITunnel API). Реализовано на Flutter (фронтенд) и FastAPI (бэкенд).

---

## Структура проекта

### Backend (Python/FastAPI)

#### `config.py`
Конфигурация приложения:
- `AITUNNEL_API_KEY` - API ключ для облачного сервиса AITunnel (интеграция с GPT-4o)
- `AITUNNEL_URL` - URL эндпоинта API
- `DB_*` - параметры подключения к PostgreSQL
- `UPLOAD_DIR` - папка для сохранения загруженных изображений

#### `main.py`
Главный файл приложения:
- Инициализация FastAPI сервера
- Регистрация роутеров (auth, recognition, search и т.д.)
- Запуск сервера на localhost:8000

#### `database.py`
Работа с базой данных:
- `get_db_connection()` - создание подключения к PostgreSQL
- Выполнение SQL запросов для работы с таблицами users, photos, ai_requests, recognition_results и т.д.

#### `routers/recognition.py` (КЛЮЧЕВОЙ ФАЙЛ)
Основная логика распознавания одежды:

**Функции:**
- `save_uploaded_file()` - сохраняет загруженное изображение на диск с уникальным именем
- `recognize_with_aitunnel()` - отправляет изображение в AITunnel API (GPT-4o vision), получает текстовое описание одежды
- `map_caption_to_item()` - преобразует текстовое описание (например "black adidas hoodie") в структурированные данные (бренд, модель, категория)
- `recognize_from_filename()` - fallback функция, если ИИ недоступен - пытается определить одежду по имени файла
- `insert_photo()`, `insert_ai_request()`, `update_ai_request_status()` - работа с БД (сохранение фото, статус запроса)
- `get_brand_id()`, `get_category_id()`, `get_model_id()` - получение ID из БД по названию
- `insert_recognition_result()` - сохранение результата распознавания в БД
- **`recognize_item()`** - главный эндпоинт, обрабатывает POST запрос с изображением:
  1. Сохраняет файл
  2. Создает запись в БД (статус "processing")
  3. Отправляет на AITunnel для распознавания
  4. Маппит результат в структурированные данные
  5. Если ИИ не ответил - использует fallback (имя файла)
  6. Сохраняет результат в БД (статус "done")
  7. Возвращает результат клиенту

**Поток данных:**
```
Изображение (Flutter) → 
  save_uploaded_file() → 
  recognize_with_aitunnel() (AITunnel API/GPT-4o) → 
  map_caption_to_item() → 
  insert_recognition_result() → 
  Результат (Flutter)
```

#### `routers/auth.py`
Аутентификация пользователей:
- `register()` - регистрация нового пользователя
- `login()` - вход в систему
- Работает с таблицей `users` в БД

#### `routers/search.py`
Поиск по истории:
- Поиск распознанной одежды по разным параметрам
- Фильтрация по бренду, категории, дате

## API Интеграция: AITunnel

### Что такое AITunnel?
Облачный сервис для доступа к различным ИИ моделям (OpenAI, Anthropic, Google и т.д.) через единый API.

### Как используется в проекте?

**В `recognition.py` функция `recognize_with_aitunnel()`:**
```
1. Берет изображение (bytes)
2. Кодирует его в Base64
3. Создает JSON payload с сообщением:
   - Тип: image_url с Base64 данными
   - Промпт: "Look at the clothing... Write only: clothing type and brand"
4. Отправляет POST запрос на https://api.aitunnel.ru/v1/chat/completions
5. Заголовок Authorization: Bearer {API_KEY}
6. Модель: gpt-4o (мультимодальная, работает с изображениями)
7. Получает ответ: "black adidas hoodie" или "nike sneakers"
8. Возвращает текст в нижнем регистре
```

**Пример запроса:**
```json
{
  "model": "gpt-4o",
  "messages": [{
    "role": "user",
    "content": [
      {"type": "text", "text": "Describe clothing..."},
      {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,..."}}
    ]
  }],
  "max_tokens": 60
}
```

## База данных (PostgreSQL)

### Основные таблицы:

**users** - данные пользователей
- user_id, username, email, password, role (admin/user)

**photos** - загруженные изображения
- photo_id, user_id, photo_path (путь к файлу)

**ai_requests** - история запросов к ИИ
- request_id, photo_id, status (processing/done/error)

**recognition_results** - результаты распознавания
- request_id, category_id, brand_id, model_id, confidence_score

**brands** - справочник брендов
- brand_id, brand_name (Adidas, Nike, Levis и т.д.)

**categories** - справочник категорий
- category_id, category_name (Худи, Кроссовки, Джинсы)

**clothing_models** - справочник моделей одежды
- model_id, model_name (ENT22 HOODY Y, Air Max 90, 501 Original)

**favorites** - избранная одежда пользователя
- favorite_id, user_id, brand_id, category_id, model_id

## Поток работы приложения

### Сценарий: Пользователь загружает фото одежды

1. **Фронтенд (Flutter):**
   - Пользователь выбирает фото из галереи или делает снимок
   - Отправляется POST запрос на `/recognize` с параметрами:
     - `user_id` - ID пользователя
     - `file` - бинарное изображение

2. **Бэкенд (FastAPI):**
   - Функция `recognize_item()` получает запрос
   - Сохраняет изображение на диск через `save_uploaded_file()`
   - Создает запись в таблице `ai_requests` со статусом "processing"
   - Отправляет изображение в AITunnel API через `recognize_with_aitunnel()`
   - Получает текстовое описание, например: "black nike hoodie"
   - Маппит описание в структурированные данные через `map_caption_to_item()`:
     - brand: "Nike"
     - model: "Air Max 90"
     - category: "Худи"
     - confidence_score: 90.0
   - Сохраняет результат в БД через `insert_recognition_result()`
   - Обновляет статус на "done"
   - Возвращает результат JSON клиенту

3. **Фронтенд (Flutter):**
   - Получает результат
   - Показывает экран `ResultScreen` с полученными данными
   - Пользователь может уточнить выбор (изменить категорию/бренд/модель)
   - Нажимает "Сохранить в избранное"
   - Функция `_saveToFavorites()` отправляет выбранные значения на бэкенд
   - Результат сохраняется в таблице `favorites`

---

## Запасной режим (Fallback)

Если AITunnel API недоступен:
1. В `recognize_item()` вызывается `recognize_from_filename()`
2. Функция анализирует имя файла
3. Если в имени есть "hoodie" → возвращает Худи от Adidas
4. Если "nike" + "shoe" → Nike Air Max 90
5. Если "jeans" или "levis" → Levis 501
6. Если ничего не совпало → возвращает default значения

Это обеспечивает стабильность приложения даже при недоступности облачного API.

## Что спросят на защите и как ответить

### "Как работает распознавание?"
Ответ: Изображение отправляется в облачный сервис AITunnel, который использует модель GPT-4o для анализа. Модель возвращает текстовое описание (например "black nike hoodie"), которое затем преобразуется в структурированные данные (бренд, модель, категория).

### "Где хранятся данные?"
Ответ: В PostgreSQL базе данных. Все загруженные изображения сохраняются на диск в папке `uploads`, а метаданные (результаты, пользователи, история) — в таблицах БД.

### "Что если API недоступен?"
Ответ: Есть fallback механизм. Если AITunnel недоступен, приложение пытается определить одежду по имени файла. Это обеспечивает стабильность.

### "Как работает сохранение в избранное?"
Ответ: На экране результатов пользователь может уточнить выбор (категория/бренд/модель), а затем нажать "Сохранить". Функция `_saveToFavorites()` отправляет выбранные значения на бэкенд, которые сохраняются в таблице `favorites`.

### "Как организована база данных?"
Ответ: Используется PostgreSQL с несколькими таблицами: users (пользователи), photos (изображения), recognition_results (результаты ИИ), brands (справочник брендов), categories (категории), favorites (избранное).

### "Почему использован AITunnel вместо OpenAI?"
Ответ: AITunnel предоставляет более гибкий доступ к различным ИИ моделям через единый API, включая GPT-4o, и работает стабильнее из России.
