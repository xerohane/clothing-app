Контекст проекта (готовый текст)

Я разрабатываю дипломный проект — систему распознавания одежды по фотографии с последующим поиском товаров в интернет-магазинах.

🔹 Общая идея

Пользователь загружает фотографию одежды (например, кроссовок, худи или джинсов), после чего система:

Анализирует изображение с помощью AI
Определяет категорию, бренд и модель одежды
Сохраняет результат в базе данных
Показывает пользователю похожие товары из магазинов с ценами и ссылками
🔹 Архитектура проекта
🧠 Backend (FastAPI, Python)
Обрабатывает загрузку изображений
Отправляет изображение в HuggingFace API (модель blip-image-captioning-base)
Получает описание изображения (caption)
Преобразует описание в:
бренд
модель
категорию
Сохраняет результат в PostgreSQL
Отдает данные во Flutter-приложение
📱 Frontend (Flutter)
Позволяет пользователю:
загрузить фото
увидеть результат распознавания
нажать "Найти в магазинах"
Отображает:
бренд
модель
категорию
список предложений из магазинов
🗄️ База данных (PostgreSQL)

Используется реляционная структура:

Основные таблицы:

users — пользователи
photos — загруженные изображения
ai_requests — запросы на распознавание
recognition_results — результаты AI
brands, categories, clothing_models — справочники
products — товары
stores — магазины
product_offers — предложения (цены, ссылки)
favorites — избранное
🔹 Логика работы системы
1. Загрузка изображения

Фото сохраняется на сервере → запись в photos

2. Создание запроса

Создается запись в ai_requests со статусом processing

3. AI-распознавание
Отправка изображения в HuggingFace
Получение текстового описания (caption)
4. Интерпретация результата

Функция map_caption_to_item() определяет:

категорию (например, "Джинсы")
бренд (например, "Levis")
модель (например, "501 Original")

Если AI не отвечает → используется fallback

5. Сохранение результата

Данные записываются в:

recognition_results
статус запроса меняется на done
6. Поиск товаров

По модели выполняется SQL-запрос:

соединение products + stores + offers
возвращаются:
цена
магазин
ссылка
